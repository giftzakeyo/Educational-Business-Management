### 2. Basic GitHub Actions

#### View Files:
- Click on folder names to navigate
- Click on file names to view content
- Use "Go to file" to search

#### Add New Files:
1. Click "Add file" → "Create new file"
2. Enter filename (e.g., `Business/Financials/april_2026.md`)
3. Add content
4. Scroll down, add commit message
5. Click "Commit new file"

#### Upload Existing Files:
1. Click "Add file" → "Upload files"
2. Drag and drop files/folders
3. Add commit message
4. Click "Commit changes"

## 💡 Best Practices

### Commit Messages:
- Be descriptive: "Added Chemistry equilibrium worksheet"
- Use present tense: "Adds", "Updates", "Creates"
- Keep them concise but informative

### File Organization:
- Use consistent naming: `YYYY-MM-DD_topic.md`
- Group related files in folders
- Update README.md when adding new sections

### Security:
- Never commit:
  - Passwords or API keys
  - Personal identification numbers
  - Financial account details
  - Student personal information
- Use `.gitignore` for sensitive files
- Consider encryption for very sensitive data
